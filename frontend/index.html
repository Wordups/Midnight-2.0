Architecture

Three clean layers.
No bleed-through.

Extraction, mapping, and rendering are strictly separated. Each layer does one job and passes clean data to the next.

01
Extraction
Reads the source document. Pulls fields, sections, bullets, tables, and revision history. Outputs structured POLICY_DATA.
02
Mapping
Normalizes extracted content into a fixed schema. Validates required fields. Preserves original wording exactly — no paraphrasing.
03
Rendering
Rebuilds into the target template. Handles layout only. No extraction logic — deterministic output regardless of input length.
